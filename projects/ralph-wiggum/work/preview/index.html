03-technique - The Technique (Diagram) 31.72s

clips/03-technique.mp4 + audio/03-technique-a.wav + audio/03-technique-b.wav

The Ralph Wiggum technique flips this completely. Named after the Simpsons character who keeps trying despite setbacks. The core idea is brilliantly simple. Put Claude in a while loop.
While true. Cat prompt dot em dee. Pipe to Claude Code. Done. The prompt never changes. But Claude sees its previous work in files and git. Each iteration it reads what it built and improves.

04-plugin - Plugin Introduction (CLI) 22.00s

clips/04-plugin-narrated.mp4 (VHS speedup 1.22x + audio)

Anthropic formalized this into an official Claude Code plugin. Use slash ralph loop with your prompt, set max iterations as a safety net, and define a completion promise.
Claude will loop until it outputs that promise or hits the limit. You can cancel anytime with slash cancel ralph.

12-best-practices - Best Practices (Diagram) 28.52s

clips/12-best-practices.mp4

Writing good ralph prompts is an art. First, clear completion criteria. What does done look like. Second, break complex tasks into phases.
Third, include self correction. Tell Claude to run tests and fix failures. Fourth, always set max iterations. It is your safety net. Failures are data for tuning your prompt.
