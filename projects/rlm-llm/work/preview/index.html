RLM Level 4: LLM-Based Analysis - Video Preview

When data exceeds context limits, delegate to sub-LLM calls and aggregate results.

Core Concept

Problem: Some datasets are too large for any LLM context window.

Solution: L1/L2/L3 extract and reduce → L4 chunks and delegates to sub-LLMs → Aggregate results.

Architecture

Input Data (too large for context) │ ▼ ┌─────────────────────────────────────┐ │ L1/L2/L3: Extract & Reduce │ └─────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────┐ │ L4: Chunk & Delegate to Sub-LLMs │ └─────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────┐ │ L4: Aggregate Chunk Results │ └─────────────────────────────────────┘ │ ▼ Final Answer

Example Use Case: War and Peace Family Tree

Input

War and Peace novel: 3.2MB, ~580K words, far exceeds any context window

L1 Phase

Extract character names, dialogue attributions, relationship mentions

L4 Chunk Phase

Split extracted data into ~8K token chunks with character context overlap

L4 Sub-LLM Phase

Each chunk analyzed for: character relationships, family connections, social dynamics

L4 Aggregation

Combine chunk analyses, resolve conflicts, build complete family tree
